RHI 弹窗四Tab改造开发方案。

点击信号分组的 RHI 按钮后弹出的窗口，从单一画布改造为四个 Tab 视图，视觉遵循 EWIS可视化规范V1.md。

## 参考文件

- **视觉规范**：`EWIS可视化规范V1.md` — 五层架构：本体→样式→布局→交互→视图
- **预览原型**：`RHI可视化预览.html` — 四Tab完整HTML原型（纯前端静态数据）
- **现有组件**：`client/src/components/RhiEditor.tsx` — 当前RHI编辑器（将成为Tab 2）

## 四个 Tab 定义

### Tab 1：EICD 逻辑连接
展示该信号组内所有设备之间的逻辑连接拓扑。

**数据来源**：
- signals + signal_endpoints + signal_edges（信号及连接关系）
- devices + connectors + pins（设备层级）

**视觉要素**：
- 设备体：主设备(`#ECFDF5`/`#059669`) vs 远端设备(`#EFF6FF`/`#2563EB`)
- 连接器：贴在设备朝向对端一侧外壁，`#EFF6FF`/`#3B82F6`，编号在设备体内竖排
- 针孔：12×12px方块贴在连接器外侧，编号在连接器内
- ATA徽章：设备框左下角
- 协议护套：虚线圆角矩形 `#A78BFA`边框 + `#F5F3FF`浅紫填充，包裹同协议的所有连线
- 绞线编织（Q1-A规范五段路径）：
  - ①Bezier引入 → ②直线延伸(20px) → ③正弦编织(z-order逐段交替) → ④直线延伸(20px) → ⑤Bezier引出
  - 双绞：蓝`#2563EB` + 红`#DC2626`，振幅5px，波长14px
  - 三绞：蓝+红+绿`#059669`，振幅5px，波长18px
  - 采样密度：每波长24点（非固定步数）
  - 编织段长度量化为波长整数倍
  - 初始相位偏移确保两端对齐
- 方向箭头动画（SVG animateMotion）：toRemote蓝、toMain琥珀、bidirectional紫
- 多端协议（≥3设备）汇流节点：环形扇形节点 + 最优旋转搜索 + 针孔重排 + 投影排序
- 护套Y坐标按源端针孔中位数排序（最小交叉）
- Bezier切线方向感知：针孔端朝外，护套端水平进入

**右侧属性面板**（260px）：点击设备/连线显示详情（设备编号、ATA、连接器、信号名等）

### Tab 2：RHI 物理布线（现有功能）
**直接迁入现有 RhiEditor.tsx 的全部功能**：
- 左侧互联点面板（按类型/区域分组、拖入画布）
- SVG画布（设备节点 + 互联点节点 + 针孔连线 + 绞线编织）
- 针孔分配对话框
- 保存/导出CSV
- 所有交互：拖拽移动、拖拽针孔连线、右键删除、点击删除连线
- RHI设备样式：填色`#FFF7ED`，边框`#D97706`，RHI徽章

### Tab 3：层级树视图
展示信号组内线束的层级结构。

**视觉要素**：
- 左侧面板（280px）：线束目录树 + 搜索框 + 全局统计
  - 树节点：线束 → 协议组 → 绞线对 → 导线
  - 可展开/折叠，点击聚焦
  - 选中高亮（紫色左边框`#7C3AED`）
- 右侧画布：树状图SVG可视化
  - 线束主干（灰色粗线条）
  - 协议组分支
  - 绞线对分叉
  - 单线延伸
- 右侧属性面板（260px）：选中线束/导线的详情
- 底部统计栏：线束数、导线数、设备数、死接头数

**数据构建**：
- 从信号组的signals + twist_group构建层级
- 协议组 = 同signal_group内按协议标识聚合
- 绞线对 = 同twist_group的信号
- 导线 = 单条信号

### Tab 4：连接矩阵
设备×设备的连接关系矩阵。

**视觉要素**：
- 表格：行/列 = 设备编号
- 单元格：连接数（有连接的绿色背景`#ECFDF5`/`#059669`，无连接灰色`#E2E8F0`）
- 有连接的单元格可点击，展开显示具体信号列表
- 固定首行首列（sticky header）
- 字号11px，紧凑排列

**数据来源**：signal_edges + signal_endpoints + devices

## 通用UI框架

### 顶栏（48px）
```
┌──────────────────────────────────────────────────────┐
│ RHI 线路设计与布局  [MBSE EWIS]    [EICD|RHI|树|矩阵] │
└──────────────────────────────────────────────────────┘
```
- 标题：15px，font-weight 600
- 橙色徽章：`#D97706` 背景白字 10px
- Tab药丸组：灰色背景`#F1F5F9` 圆角8px，active白底+阴影
- 关闭按钮在最右

### 画布通用
- 背景：`#F8FAFC` + 点阵网格（20px间距，`#E2E8F0` 1px圆点）
- 缩放/平移：滚轮缩放 + 拖拽平移（非节点区域）
- 底部浮动工具栏：缩放+/- + 百分比显示 + 重置 + 刷新
- 图例：右上角浮动，半透明白底 + backdrop-blur

## 需要的新后端API

### Tab 1 数据（EICD逻辑连接）
现有API已足够：
- `GET /api/rhi/:signal_group?project_id=` — 返回nodes+slots+links
- 需补充：信号的edge方向信息（signal_edges.direction）、设备类型（main/remote需要前端判断或后端增加）

### Tab 4 数据（连接矩阵）
可能需要新API或在前端从Tab 1数据构建：
- 设备×设备的连接计数
- 信号组内所有edge的from_device→to_device聚合

## 开发阶段

### 阶段1：弹窗框架 + Tab切换
- RhiEditor.tsx 顶部加4个Tab按钮
- Tab 2 放现有画布内容（不改动）
- Tab 1/3/4 先放占位内容
- 遵循CSS变量和顶栏样式

### 阶段2：Tab 1 — EICD逻辑连接（核心工作量最大）
分步实现：
1. 设备体 + 连接器 + 针孔渲染（参考规范2.1-2.3）
2. 协议护套 + 单线连接（参考规范2.5）
3. 绞线Q1-A编织（五段路径 + z-order）（参考规范2.4）
4. 方向箭头动画（参考规范2.8）
5. 多端协议汇流节点（参考规范1.13 + HTML原型）
6. 属性面板 + 图例
7. 缩放/平移 + 工具栏

### 阶段3：Tab 3 — 层级树视图
1. 左侧目录树（搜索+展开折叠）
2. 右侧树状图SVG
3. 属性面板 + 统计栏

### 阶段4：Tab 4 — 连接矩阵
1. 矩阵表格渲染
2. 单元格点击展开

### 阶段5：打磨
1. Tab间数据联动（如Tab 1选中信号 → Tab 2高亮对应连线）
2. 信号流向筛选下拉（HTML原型左上角的筛选器）
3. 性能优化（大量SVG元素时的虚拟化）

## 设计红线（不可违反）

1. **绞线必须用Q1-A编织**：逐段z-order，不能用简单正弦叠加
2. **采样密度按波长**：每波长24点，不能固定步数
3. **编织段长度量化为波长整数倍**：保证两端对齐
4. **Bezier切线方向感知**：针孔端朝外，护套端水平
5. **每根导线从靠近己方设备侧进入护套**：不绕到对面
6. **护套按Y中位数排序**：最小交叉
7. **多端协议用环形汇流节点**：不用点对点
8. **设备间不允许直连**（Tab 2）：必须通过互联点
9. **绞线统一配色**：双绞红蓝，三绞红蓝绿
10. **所有连线必须归入护套**（Tab 1）：无signalGroup的也画空标签护套
